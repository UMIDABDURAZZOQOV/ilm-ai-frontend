 *AI  Engineer  vs  AI  Software  Engineer  —  What's  the  difference?*   These  two  terms  are  often  used  interchangeably,  but  there's  a  real  distinction  in  
focus
 
and
 
depth.
  ──────────────────────   
 *AI  Engineer*  (newer  role)  Focuses  on  *building  applications  on  top  of  existing  AI  models*  —  not  building  the  
models
 
themselves.
  Tech  stack:  -  LLM  APIs  (OpenAI,  Anthropic,  Gemini,  Mistral)  -  LangChain,  LlamaIndex  (orchestration)  -  Vector  databases  (Pinecone,  ChromaDB,  Qdrant)  -  Prompt  engineering  &  RAG  pipelines  -  Python,  FastAPI,  basic  cloud   Key  skills:  prompt  design,  RAG,  agent  architectures,  ﬁne-tuning  via  APIs,  evals   ──────────────────────   
 *AI  Software  Engineer*  (deeper,  broader)  Focuses  on  *building  and  training  the  underlying  ML  systems*  at  scale.   Tech  stack:  -  PyTorch,  TensorFlow,  JAX  -  Hugging  Face  (Transformers,  PEFT)  -  MLﬂow,  Weights  &  Biases  -  Kubernetes,  Docker,  Ray,  Spark  -  CUDA  /  GPU  optimization  -  Cloud  ML  (SageMaker,  Vertex  AI,  Azure  ML)  

 Key  skills:  model  architecture,  distributed  training,  data  pipelines,  benchmarking   ──────────────────────   
 *Quick  comparison*   AI  Engineer  →  builds  *on  top  of*  AI  models  AI  Sof.  Engineer  →  builds  *the*  models   AI  Engineer  →  prompt  engineering  is  core  AI  Sof.  Engineer  →  deep  ML  theory  is  core   AI  Engineer  →  lower  entry  barrier  AI  Sof.  Engineer  →  higher  entry  barrier   ──────────────────────   In  short:  *AI  Engineer*  =  product/application  builder  using  AI  as  a  tool  *AI  Software  Engineer*  =  systems  builder  working  deep  in  the  ML  stack

 *The  split  he  predicts*   Karpathy  says  AI  is  sorting  engineers  into  two  camps:   -  *Builders*  —  lean  into  AI  tools,  orchestrate  agents,  use  language  as  their  primary
